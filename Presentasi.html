Mengapa Cache Penting?

Cache mempercepat akses memori dengan menyimpan data yang sering diakses lebih dekat ke CPU, sehingga mengurangi waktu tunggu instruksi.

Tujuan Presentasi

  • Menjelaskan konsep cache memory
  • Memahami struktur direct-mapped cache
  • Mengukur hit rate dan miss rate
  • Menunjukkan pembagian alamat memori
  • Membandingkan tag, index, dan offset
  • Menjelaskan logika hit dan miss

Manfaat Cache

  • Mengurangi waktu akses ke data
  • Meningkatkan performa sistem
  • Meminimalkan beban memori utama
  • Mendukung eksekusi instruksi lebih cepat

Penjelasan Cache

Alamat memori dibagi menjadi tag, index, dan offset untuk mencari data di cache. Setiap baris cache menyimpan tag dan valid bit untuk mengecek kecocokan.

Konfigurasi cache direct-mapped

Direct-Mapped Cache

Arsitektur Direct-Mapped Cache

Analisis Arsitektur

Direct-mapped cache menetapkan satu lokasi cache tunggal untuk setiap blok memori. Setiap alamat hanya bisa berada di satu baris cache.

Struktur alamat: [tag][index][offset]

Tag Index Offset
5 bit 3 bit 2 bit

Fungsi Setiap Field

  • Tag mengidentifikasi blok memori
  • Index memilih baris cache
  • Offset menunjuk byte dalam blok

Catatan Penting

Jika tag tidak cocok, cache harus memuat ulang blok dari memori utama. Konflik cache terjadi jika beberapa alamat ingin mengisi baris yang sama.

Analisis Hasil Simulasi

Tiga Jenis Miss pada Cache

COMPULSORY MISS

Terjadi saat data pertama kali diakses karena cache masih kosong.

CONFLICT MISS

Terjadi karena beberapa address memiliki index yang sama sehingga saling menggantikan pada line cache yang sama.

CAPACITY MISS

Tidak terlalu dominan karena jumlah data masih berada dalam kapasitas cache.

Kesimpulan Simulasi

Pola address yang sering menggunakan index sama menyebabkan data lama terus tergantikan. Akibatnya conflict miss meningkat dan hit rate menjadi rendah. Pada simulasi ini hanya terdapat 1 hit dan 11 miss sehingga hit rate hanya sekitar 8%.
